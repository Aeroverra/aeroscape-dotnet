## Bugs

- `AeroScape.Server.Network/Protocol/PacketDecoders.cs:549` — `ItemOption1Decoder` reads the first 4 bytes as a packed dword, which swaps the Java field order entirely. Java reads `readUnsignedWordBigEndianA()` for `itemSlot`, then `readUnsignedWord()` for `interfaceId`, then another `readUnsignedWord()` junk field, then `readUnsignedWord()` for `itemId`, plus a trailing `readUnsignedWordBigEndian()` read. The C# decoder therefore assigns `interfaceId` from the encoded slot bytes, assigns `slot` from the junk field, and skips the trailing word. — Java reference: `legacy-java/server508/src/main/java/DavidScape/io/packets/ItemOption1.java:26`

- `AeroScape.Server.Network/Protocol/PacketDecoders.cs:643` — `ItemOption2Decoder` has the same byte-order bug as `ItemOption1Decoder`: Java reads `itemSlot` with `readUnsignedWordBigEndianA()`, then `interfaceId`, then a junk word, then `itemId`. The C# decoder instead treats the first 4 bytes as a packed interface/button dword and shifts every field two bytes off. — Java reference: `legacy-java/server508/src/main/java/DavidScape/io/packets/ItemOption2.java:16`

- `AeroScape.Server.Network/Protocol/PacketDecoders.cs:458` — `ObjectOption2Decoder` does not match the legacy Java handler at all. The Java counterpart reads only two plain `readUnsignedWord()` values on this path, while the C# decoder reads three fields with `readUnsignedWordBigEndian()`, `readUnsignedWord()`, and `readUnsignedWordBigEndianA()`. That is a different byte order, different transforms, and different total bytes consumed. — Java reference: `legacy-java/server508/src/main/java/DavidScape/io/packets/ObjectOption2.java:29`

- `AeroScape.Server.Network/Protocol/PacketDecoders.cs:580` — `MagicOnNPCDecoder` only reads one `readSignedWordA(), readSignedWordA(), readUnsignedWord()` triplet. The legacy Java flow for opcode `24` consumes that same triplet twice: first inside `MagicOnNPC.magicAop(...)`, then again in `PacketManager` before the `interfaceId == 192` check. The C# decoder is therefore missing the second 6-byte read sequence. — Java reference: `legacy-java/server508/src/main/java/DavidScape/io/packets/MagicOnNPC.java:126`, `legacy-java/server508/src/main/java/DavidScape/io/PacketManager.java:641`
